Author: CodePaul Date: 2019-10-13  16:51:00

String类

String类：代表字符串。Java 程序中的所有字符串字面值（如 “abc” ）都作为此类的实例实现。

String是一个final类，代表不可变的字符序列。

字符串是常量，用双引号引起来表示。它们的值在创建之后不能更改。

String对象的字符内容是存储在一个字符数组value[]中的。（jdk1.8）

String s1 = new String(“hello”);

字符串常量池中有一个对象，堆中有一个字符串对象。

String s1 = “hello”;

在常量池中创建对象

String s1 = new String(“hello”);

String s2 = new String(“hello”);

String s1 = “hello”;

String s2 = “hello”;

StringBuffer类

StringBuffer类是可变字符序列，在进行内容的添加和删除是不会创建新的字符对象。

构造器

StringBuffer的构造器不同于String。

  • StringBuffe() 构造一个其中不带字符的字符串缓冲区，起始容量为16个字符
  • StringBuffer(int size) 构造指定容量的字符串缓冲区
  • StringBuffer(String str) 将内容初始化为指定字符串内容

StringBuffer常用方法

  • 增 ： append(xxx) 将xxxx拼接
  • 删 ：delete(int start， int end)移除此序列的子字符串中的字符。
  • 改：replace(int start, int end)使用给定 String 中的字符替换此序列的子字符串中的字符。
  • 查：charAt(int index) 返回此序列中指定索引处的 char 值。
  • 插：insert(int offset, xxx) 在指定序列插入xxx
  • reverse() 将字符序列反转输出。

StringBuilder

StringBuilder 和 StringBuffer 非常类似，均代表可变的字符序列，而且提供相关功能的方法也一样

String，StringBuffer， StringBuilder，

String（jdk1.0）：不可变序列。

StringBuffer（jdk1.0）: 可变序列，线程安全，效率低。

StringBuilder（jdk5.0）：可变序列，线程不安全，效率高。

在作为参数传递时，方法内部String不会被改变值，StringBuffer和StringBuilder会被改变值
