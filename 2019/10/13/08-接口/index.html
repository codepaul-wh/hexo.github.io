接口

Author: CodePaul Date: 2019-10-13  19:18:00

接口不是类，接口是一种引用数据类型,是方法的容器。

使用interface关键字修饰。

接口类名包含：

  • 自定义常量
  • 抽象方法
  • 默认方法（jdk8.0）
  • 静态方法（jdk8.0）
  • 私有方法（jdk9.0）

定义格式：使用interface关键字

实现格式：使用implements关键字

默认方法（jdk8.0）

当所有的子类都有同一个方法，方法声明一样，方法内容一样，无需在子类中重复定义。

接口中，有多个默认方法时，实现类都可继承使用。如果默认方法有重名的，必须重写一次。

注意事项：

  • 在多实现的情况下，如果多个接口中有同名方法，需要重写该方法。

接口注意事项

- 当一个类实现接口类后，必须重写所有的抽象方法。

  • 当一个类继承父类同时实现多个接口时，父类中方法名和接口中的方法名相同时，子类会优先调用父类中的方法

  • 接口中只有五种成员，接口不能被实例化，因为接口中没有构造方法

  • 接口，类之间的关系 ：

    • 类和类的关系： 单继承。
    • 类和接口的关系：多实现。
    • 接口和接口的关系：多继承。

在Java 9+版本中，接口的内容可以有：

  1. 成员变量其实是常量，格式：
    [public] [static] [final] 数据类型 常量名称 = 数据值;
    注意：
    常量必须进行赋值，而且一旦赋值不能改变。
    常量名称完全大写，用下划线进行分隔。

  2. 接口中最重要的就是抽象方法，格式：
    [public] [abstract] 返回值类型 方法名称(参数列表);
    注意：实现类必须覆盖重写接口所有的抽象方法，除非实现类是抽象类。

  3. 从Java 8开始，接口里允许定义默认方法，格式：
    [public] default 返回值类型 方法名称(参数列表) { 方法体 }
    注意：默认方法也可以被覆盖重写

  4. 从Java 8开始，接口里允许定义静态方法，格式：
    [public] static 返回值类型 方法名称(参数列表) { 方法体 }
    注意：应该通过接口名称进行调用，不能通过实现类对象调用接口静态方法

  5. 从Java 9开始，接口里允许定义私有方法，格式：
    普通私有方法：private 返回值类型 方法名称(参数列表) { 方法体 }
    静态私有方法：private static 返回值类型 方法名称(参数列表) { 方法体 }
    注意：private的方法只有接口自己才能调用，不能被实现类或别人使用。

接口和抽象类

接口：
  • 接口中没有构造方法
  • 接口中没有this关键字
  • 接口中的抽象方法可以省略abstract关键字。
  • 接口可以多实现
  • 接口不是类，是一种数据类型。
  • 接口中有常量，用 public static final ，可省略，通过接口名.常量名调用
  • 接口中没有普通方法。
抽象类：
  • 抽象类中有构造方法。
  • 抽象类中有this关键字。
  • 抽象类中的抽象方法必须写abstract关键字。
  • 抽象类只能单继承。
  • 抽象类有自己的属性和方法。
  • 抽象类有普通方法。

相同点：

接口和抽象类都不可以被实例化。

都含有抽象方法，包含方法的声明，没有方法体。
